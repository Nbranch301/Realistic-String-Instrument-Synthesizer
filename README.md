After I realised that implementing synthesis in a framework such as JUCE is essentially the reverse of digitally sampling an audio source, I became eager to explore the limits of acoustic sound replication. I chose to focus on the Karplus-Strong algorithm and created parameters to alter the sound source it emulates.

Using concepts from a delay effect plugin I recently developed, I programmed a circular buffer with a read pointer i and a write pointer j. When a MIDI note begins, i points to the start of the buffer, which is filled with p values within range -1 to 1 (white noise) and j is set to position p. The value of p depends on the inputted frequency. As each block is rendered, buffer values are determined recursively using the KS algorithm. 

After implementing the base model, I added several factors to change the sound. For instance, the pick sharpness parameter ps emulates the difference between a sharp pick (ps = 1) and a rounded pick or fingers (ps = 0). In the program, this coefficient mixes between the unaltered wave function g and an altered function g2, where g2 (t) = g (t) + |g (t)|30 for any sample t. Since g (t) is strictly bounded by -1 and 1, increasing ps will not cause peaking in the waveform and will instead increase its non-linear. Thus, higher partials become more present and the fundamental dissipates, approximating the spectral changes that occur when switching picking techniques on an acoustic instrument.

Similarly, the brightness parameter controls the strength of the low-pass filter used in the recurrence. I chose this name since a longer decay in higher harmonics is generally perceived as brighter. While this does contribute to note decay and resonance, I also added ADSR controls for more amplitude shaping control over time. Additionally, I introduced a parameter c, representing the max pitch sharpening in cents, depending on velocity. The detune factor for each note is randomized; however, it will generate within a range of 0 to c with an input at maximum velocity and will not detune at all at minimum velocity. This mimics the way certain string instruments will sound slightly sharper when plucked harder.

In addition to user controls, the plugin interface includes several instrument presets. My implementation uses one high-pass and four band filters whose frequency and gain values are hard-coded for each instrument based on their common resonant frequencies (along with values for the parameters mentioned above). The final knob parameter, body colour, multiplies all the filters’ gain values simultaneously, allowing the user to adjust the overall strength of resonant frequencies.

Many of my technical decisions were made after researching Karplus & Strong and Jaffe & Smith’s work. I approached the project as a hands-on way to understand their ideas and build on them. My goal was to think critically about the characteristics of string instruments and apply them creatively in the implementation process. Overall, I am satisfied with the outcome.
